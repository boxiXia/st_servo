# STS3215 Smart Bus Servo Communication Protocol

**Sources:** - `sts3215_memory_table.xlsx` (Firmware V2.43 / V3.6)
- `Smart Bus Servo Communication Protocol Manual.pdf`

## 1. Overview

The STS3215 uses a half-duplex asynchronous serial communication protocol. It supports daisy-chaining up to 253 servos on a single bus.

* **Baud Rate:** Adjustable (Default: 1,000,000 bps)
* **ID Range:** `0`–`253` (`0x00`–`0xFD`).
* **Broadcast ID:** `254` (`0xFE`). Messages sent to this ID are executed by all servos but generate no reply.
* **Endianness:** Little Endian (Low Byte first, High Byte second).
* **Storage Types:**
    * **EPROM:** Non-volatile. Values persist after power cycle. (Requires "Unlock" to write).
    * **SRAM:** Volatile. Values reset to default/initial states after power cycle.

---

## 2. Packet Structure

### 2.1 Instruction Packet (Controller → Servo)

| Byte 0 | Byte 1 | Byte 2 | Byte 3 | Byte 4 | Byte 5 ... N | Byte N+1 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `0xFF` | `0xFF` | **ID** | **Length** | **Instruction** | **Parameters** | **Checksum** |

* **Length:** Number of Parameters + 2 (Instruction + Checksum).
* **Checksum:** `~(ID + Length + Instruction + Parameters) & 0xFF`.

### 2.2 Reply Packet (Servo → Controller)

| Byte 0 | Byte 1 | Byte 2 | Byte 3 | Byte 4 | Byte 5 ... N | Byte N+1 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `0xFF` | `0xFF` | **ID** | **Length** | **Status** | **Parameters** | **Checksum** |

* **Status:** `0` indicates success. Bits indicate specific errors (see Section 5).

---

## 3. Instruction Set

| Value | Mnemonic | Description |
| :--- | :--- | :--- |
| `0x01` | **PING** | Query servo presence and status. |
| `0x02` | **READ** | Read data from the control table. |
| `0x03` | **WRITE** | Write data to the control table (Immediate execution). |
| `0x04` | **REG WRITE**| Write data to buffer (Wait for ACTION to execute). |
| `0x05` | **ACTION** | Trigger execution of buffered REG WRITE commands. |
| `0x06` | **RESET** | Reset control table to factory defaults. |
| `0x0A` | **CLEAR TURNS** | Reset the multi-turn counter (Special command). |
| `0x83` | **SYNC WRITE**| Write to multiple servos at once (Broadcast only). |

## 5. Detailed Examples

### 5.1 Basic Setup & Configuration

**1. Unlock EPROM (Disable Write Protection)**
To modify ID or Baud Rate, you must unlock the servo first.
* **Packet:** `FF FF 01 04 03 37 00 C0`
    * ID: `01`
    * Instr: `03` (Write)
    * Addr: `37` (55)
    * Data: `00` (Unlock)

**2. Change ID (Broadcast Mode)**
Change all connected servos (Broadcast ID `FE`) to ID `02`.
* **Packet:** `FF FF FE 04 03 05 02 F3`
    * ID: `FE` (Broadcast)
    * Instr: `03` (Write)
    * Addr: `05` (ID Register)
    * Data: `02` (New ID)

**3. Modify Overload Protection**
Enable protection features by writing to "Unloading Condition" (Addr `0x13`).
* **Packet:** `FF FF 01 04 03 13 20 C4`
    * Addr: `13` (19)
    * Data: `20` (32 decimal - enables specific bit)

**4. Set Current Position as Center (Correction)**
Write `128` (`0x80`) to the Torque Switch address (`0x28`). This sets the current physical position to be read as step 2048.
* **Packet:** `FF FF 01 04 03 28 80 4F`
    * Addr: `28` (40)
    * Data: `80` (128)

**5. Enable/Disable Torque**
* **Enable:** Write `1` to Addr `0x28`.
* **Disable:** Write `0` to Addr `0x28`.

---

### 5.2 Motion Control

**1. Position Control (ST Series)**
Move ID `1` to Position `1000` (`0x03E8`) with Acceleration `50` (`0x32`) and Speed `1000` (`0x03E8`).
* **Sequence:** Write 6 bytes starting at Address `0x29` (Acceleration).
* **Data:** `[Accel] [Pos_L] [Pos_H] [Time_L] [Time_H] [Speed_L] [Speed_H]` (Note: Time is set to 0 here).
* **Packet:** `FF FF 01 0A 03 29 32 E8 03 00 00 E8 03 C0`

**2. Wheel Mode (Continuous Rotation)**
* **Step 1:** Write `1` to Address `0x21` (Operation Mode).
* **Step 2:** Write Speed to Address `0x2E` (Running Speed).
    * Bit 15 of Speed controls direction (0 = CW, 1 = CCW).
    * Example: Speed 1000 CW.
    * **Packet:** `FF FF 01 05 03 2E E8 03 DC`

---

### 5.3 Advanced Instructions

**1. Read Current Position**
Read 2 bytes from Address `0x38` (56).
* **Packet:** `FF FF 01 04 02 38 02 BE`
* **Reply:** `FF FF 01 04 00 18 05 DD`
    * Value: `05 18` -> `0x0518` -> `1304` steps.

**2. Clear Multi-Turn Counter**
Resets the multi-turn accumulation.
* **Packet:** `FF FF 01 02 0A F2`
    * Instr: `0A` (Special "Clear" instruction).

**3. Asynchronous Write (REG WRITE + ACTION)**
Move ID `1` and ID `2` simultaneously.
* **Step 1:** Send **REG WRITE** (`0x04`) to ID `1`.
    * `FF FF 01 09 04 2A 00 08 00 00 E8 03 D4` (Target: 2048, Speed: 1000).
* **Step 2:** Send **REG WRITE** (`0x04`) to ID `2`.
    * `FF FF 02 09 04 2A 00 08 00 00 E8 03 D3`
* **Step 3:** Send **ACTION** (`0x05`) to Broadcast (`FE`).
    * `FF FF FE 02 05 FA` (Both servos start moving exactly now).

**4. Synchronous Write (SYNC WRITE)**
Set Position `0x0800` (2048), Time `0`, Speed `0x03E8` (1000) for IDs `1`, `2`, `3`, `4` in one packet.
* **Instruction:** `0x83`
* **Address:** `0x2A`
* **Data Length per Servo:** `6` bytes.
* **Packet:**
    ```
    FF FF FE 20 83 2A 06    (Header, ID, Len=32, Instr, Addr, DataLen)
    01 00 08 00 00 E8 03    (ID 1 + 6 bytes data)
    02 00 08 00 00 E8 03    (ID 2 + 6 bytes data)
    03 00 08 00 00 E8 03    (ID 3 + 6 bytes data)
    04 00 08 00 00 E8 03    (ID 4 + 6 bytes data)
    58                      (Checksum)
    ```

## 6. Status Error Bits
The "Status" or "Error" byte (Byte 4 in Reply Packet, or Address `0x41`) contains the following flags:

| Bit | Name | Description |
| :--- | :--- | :--- |
| **0** | Input Voltage | Voltage exceeded Min/Max limits. |
| **1** | Angle Limit | Target/Current angle outside limits. |
| **2** | Temperature | Overheating (Exceeded Max Temp). |
| **3** | Range | Send parameter is out of range. |
| **4** | Mode | Checksum error or Invalid command. |
| **5** | Overload | Load exceeded "Overload Torque" for "Protection Time". |
